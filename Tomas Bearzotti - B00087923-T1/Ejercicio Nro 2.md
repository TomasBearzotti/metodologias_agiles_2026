# Ejercicio Nro: 2

## Enunciado
¿Qué es para vosotros el desarrollo de software?
Vamos a contestar entre todos algunas preguntas, para conocer vuestras opiniones y experiencias:
* ¿Ejemplos de desarrollo de software en los que hayas participado? ¿Ejemplos que conozcas?
* ¿Cuál es el papel del desarrollador en el desarrollo de un proyecto software?
* ¿Hay otros papeles importantes en el desarrollo, aparte del desarrollador?
* ¿Cuándo se ha desarrollado un buen software?

## Resolución

Para mí el desarrollo de software es todo el proceso de crear un programa o sistema para resolver un problema concreto. No es solo sentarse a programar, sino entender qué se necesita, diseñarlo, escribir el código, probarlo y después mantenerlo funcionando.

- Sobre ejemplos en los que participé, estuve en proyectos de la facultad con un sistema web para gestionar torneos deportivos usando herramientas como Node y TypeScript. Ejemplos que conozco hay en todos lados, desde cualquier app del celular hasta los sistemas operativos.
- El papel del desarrollador va más allá de escribir código. Tiene que entender el requerimiento que le plantean, pensar la mejor forma de resolverlo a nivel técnico, estructurar bien los datos y asegurarse de que lo que hace cumpla con lo que se pidió.
- Hay otros papeles muy importantes. Están los que diseñan la interfaz para que sea fácil de usar, los testers que se encargan de buscar errores antes de que salga a producción, los administradores de sistemas o servidores, y los que gestionan el proyecto para organizar al equipo.
- Creo que se desarrolló un buen software cuando hace lo que el usuario necesita de forma intuitiva, no se cuelga, funciona fluido, y el código por detrás está lo suficientemente ordenado como para poder agregarle cosas nuevas en el futuro sin que se rompa todo.
