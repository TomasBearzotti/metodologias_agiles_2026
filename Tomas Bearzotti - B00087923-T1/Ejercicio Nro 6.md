# Ejercicio Nro: 6

## Enunciado
Brindar un ejemplo en que se vean reflejadas las siguientes ventajas de la aplicación de normas en el desarrollo de software:
- Mejora la calidad del software: Reduce el número de errores y defectos, aumenta la confiabilidad y la seguridad.
- Optimiza el proceso de desarrollo: Aumenta la eficiencia y la productividad.
- Mejora la satisfacción del cliente: Garantiza que el software cumple con las expectativas del cliente.
- Reduce costos: Minimiza los costos de desarrollo y mantenimiento.
- Aumenta la competitividad: Permite a las empresas diferenciarse de la competencia.

## Resolución

Un buen ejemplo es cuando en un equipo decidimos aplicar un estándar de código estricto y convenciones claras para subir los cambios al repositorio.

Al hacer esto la calidad del software mejora porque las herramientas automáticas detectan errores antes de compilar. El proceso de desarrollo se optimiza ya que todos programan con el mismo estilo y no se pierde tiempo discutiendo formatos. Como el sistema falla menos, la satisfacción del cliente mejora al recibir un producto más estable. Se reducen los costos porque arreglar un error apenas se escribe el código lleva mucho menos tiempo que arreglarlo cuando el sistema ya está funcionando. Finalmente, esto aumenta la competitividad porque el equipo puede sacar nuevas funcionalidades más rápido y con mejor calidad que otros.
